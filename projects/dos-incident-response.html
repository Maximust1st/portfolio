Incident Response

Denial-of-Service (DoS) Incident Response & NIST CSF Analysis

This case study demonstrates the application of the NIST Cybersecurity Framework to analyze and improve organizational defenses following a Denial-of-Service (DoS) attack caused by an ICMP flood.

← Back to Projects

Scenario

Incident Scenario

As a cybersecurity analyst working for a multimedia company that provides web design, graphic design, and social media marketing services, I analyzed a simulated Denial-of-Service (DoS) incident that disrupted internal business operations.

The organization experienced an ICMP flood attack that caused internal network services to become unavailable for approximately two hours. Legitimate traffic could not reach network resources because excessive ICMP packets overwhelmed the network.

The incident response team blocked incoming ICMP traffic, took non-critical services offline, and restored critical business services. A later investigation discovered that the attack entered through an improperly configured firewall, allowing the attacker to overwhelm the network.

Security Improvements

Controls Implemented

  • Firewall rule to rate-limit incoming ICMP packets
  • Source IP verification to detect spoofed addresses
  • Network monitoring software for abnormal traffic detection
  • IDS/IPS solution to filter suspicious ICMP traffic

Framework

Applying the NIST Cybersecurity Framework

The incident was analyzed using the five core functions of the NIST Cybersecurity Framework: Identify, Protect, Detect, Respond, and Recover. This structured approach helps organizations reduce cybersecurity risk and improve incident readiness.

Identify

Identify Security Risks

  • Improperly configured firewall allowed ICMP flood traffic
  • Internal network resources became unavailable
  • Business operations were disrupted for approximately two hours
  • Firewall configuration management required improvement

Protect

Protection Strategy

  • Rate-limit ICMP packets at the firewall
  • Enable source IP validation
  • Deploy IDS/IPS protection
  • Conduct periodic firewall audits
  • Review network security policies regularly

Detect

Detection Improvements

  • Deploy continuous network monitoring
  • Enable IDS alerts for ICMP floods
  • Monitor firewall logs for anomalies
  • Utilize SIEM tools for faster alerting

Respond

Incident Response

  • Blocked malicious ICMP traffic
  • Stopped non-critical network services
  • Restored essential business services
  • Analyzed attack vectors and firewall weaknesses
  • Documented lessons learned

Recover

Recovery Plan

  • Restore affected services safely
  • Verify firewall configuration changes
  • Continue monitoring for repeated attacks
  • Update incident response documentation
  • Improve future disaster recovery planning
